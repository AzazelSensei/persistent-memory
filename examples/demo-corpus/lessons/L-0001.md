# `ALTER TABLE ... ADD COLUMN DEFAULT` locked the orders table in production

## What happened
A migration adding a column with a non-null default to the orders table took an ACCESS EXCLUSIVE lock and blocked writes for 40 seconds during peak traffic.

## Why
On the Postgres version in production, adding a column with a default rewrites the whole table. The migration had only been tested against an empty development database.

## When discovered
During the 2026-01-18 deploy, when order creation alerts fired.

## General rule
Test migrations against a production-sized dataset, and split "add column" and "backfill default" into separate steps on large tables.
