# Event-driven cache invalidation replaces fixed TTL

## Context / Problem
The fixed 5-minute TTL (D-0001) served stale prices during flash sales, causing checkout mismatches between the listing page and the cart.

## Decision
Invalidate Redis keys from a post-commit hook on price and inventory writes; keep a 1-hour TTL only as a safety net.

## Rationale
Staleness is now bounded by replication lag instead of a wall-clock window. The write path gains one hook, which is acceptable next to the cost of wrong prices at checkout.

## Outcome / Learned
Checkout mismatches dropped to zero in the first week. The old record stays visible with its reasoning — decisions are never deleted, only superseded.
